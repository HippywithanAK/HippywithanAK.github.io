Javascript, The DOM and Devtools

Here is an analogy of the differences between HTML and CSS

HTML is like an ingredients list. It tells your browser which things need to appear on a web page. It doesn't tell it what to do with those ingredients though.

CSS is like a recipe. It takes the ingredients list and gives directions for how they should be used. Just like a recipe might tell you: the garlic in your ingredients list needs to be crushed and sauted at the start of cooking, CSS might tell the browser: the header element needs to have a blue background, a red border and sit at the top of the page.

Control Flow and Loops

Loops are peices of code that repeate a certain task, untill or while, a condition is met.

Control flow describes the order that code is evaluated in.

A real life analogy for these concepts, might be cooking. Lets say you are cooking a recipe and it asks you to whisk eggs. It will tell you to do so untill they have reached a certain constency. It won't say "wisk, check consistency, if consistency is good: stop, otherwise: whisk, check consistency" over and over again. It assumes you will understand that you need to do this. A computer does not understand anything, it does what it's told. So you would use a loop to tell it to repeate "whisk, check consistency" untill the consistency is good.

This would look something like:

while (eggs.consistency != "soft peaks") {
whisk();
}

This will check if the consistency property of object eggs is equal to "soft peaks". If not, it will perform the whisk function. Otherwise it will do nothing.

Now normally, we would first need to crack eggs into a bowl before whisking. Control flow makes sure the eggs are cracked befor they are wisked. Lets whisk a whole carton:

for (i=0); i < eggCarton.length; i++){
crackInBowl(eggCarton[i])
}

Now to put it all together, making sure our eggs are all cracked into a bowl then whisked.

function wiskEggs () {
for (i=0); i < eggCarton.length; i++){
crackInBowl(eggCarton[i])
}
while (eggs.consistency != "soft peaks") {
whisk();
}
}

Code is evaluated left to right, top to bottom. Therefore the eggs cracking 'for' loop will run before the whisking 'while' loop. This gets a little more complicated when nesting loop functions. Basically the outer loop is run once, first, then the inner loop runs once. This repeats untill all itterations are complete.

Accessing Data: Arrays Vs. Objects

The Data in an array is accessed by giving it an index value:

var foo = arr[0]

will assign the first item in the arr array to the variable, foo


The data in an object is accessed using key : value pairs:

var foo = obj['name']

will look for a property with a key of 'name' in the object, obj. It then assigns the corresponding value to the variable, foo.

Functions and why they're usefull

A function is code we can run repeatedly. For example: if I want to multiply 100 by 2 and 50 by 2, I can something like

var doubleHundred = 100 * 2
var doubleFifty = 50 * 2

Or I could right a function that will take any number I like and double it:

function double = (num) {
var doubled = num * 2;
return doubled;
}

Now if I call double(someNumber) in my code, what ever 2 x someNumber is will be returned.
